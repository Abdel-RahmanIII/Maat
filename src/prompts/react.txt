Available tools:
- validate_move(fen, move_uci): The only rule-enforcement tool. Returns legal/reason/rule_ref.
- is_in_check(fen): Returns whether side-to-move is in check and checking_squares.
- get_game_phase(move_history): Returns opening | middlegame | endgame from ply count.
- get_move_history_pgn(move_history): Converts UCI move history to readable PGN text.
- get_board_visual(fen): Returns ASCII 8x8 board. (fen mode only)
- get_piece_at(fen, square): Returns piece code such as wN or empty. (fen mode only)
- get_attackers(fen, square): Returns pieces attacking the square. (fen mode only)
- get_defenders(fen, square): Returns defenders of the square occupant. (fen mode only)
- is_square_safe(fen, square, color): Returns safety boolean and threats. (fen mode only)
- get_position_after_moves(fen, moves): Applies move sequence and returns resulting FEN. (fen mode only)
- submit_move(uci_move): Submit your final move. Call this ONLY when you are confident.

In history mode, fen-only tools are unavailable and board state is withheld.

IMPORTANT: You must eventually call submit_move to play. Do not just describe actions; call tools when needed. Think carefully, validate if unsure, then submit.
